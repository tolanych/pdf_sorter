Goal: generate a meaningful file name based on the file's content.

Naming rules:
1) Language: English, Latin alphabet, snake_case.
2) Preferred format: <category>_<topic>_<date>.
3) Example categories: invoice, tax_return, vat, social_insurance, bank_statement, residence_permit, contract, certificate, power_of_attorney, application, photo, scan, report.
4) If date is known — YYYY-MM or YYYY-MM-DD; otherwise omit the date.
5) No random hashes/IDs unless it is an official document number.
6) The name should be short, clear, and unique within the folder.
7) Do not change the file extension.
8) If content is unreadable — use: unreadable_scan.

Return JSON with fields: title,type,summary,date,confidence.
